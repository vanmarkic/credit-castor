# Documentation

This directory contains project documentation generated and maintained by Claude Code.

## Directory Structure

### `development/`
Implementation plans, progress tracking, and feature documentation:
- Implementation plans from `/superpowers:write-plan`
- Feature specifications
- Technical design documents
- Progress updates

### `analysis/`
Code analysis, architecture reviews, and quality assessments:
- Architecture analysis
- Code review reports
- Business logic separation audits
- Performance analysis
- Security reviews

### `history/`
Decision logs and retrospectives:
- ADRs (Architecture Decision Records)
- Retrospectives
- Migration notes
- Lessons learned

## Guidelines

- Update existing documents instead of creating duplicates
- Use clear, descriptive filenames (e.g., `feature-user-auth-plan.md`)
- Include dates in filenames for time-sensitive content (e.g., `2025-03-sprint-retrospective.md`)
- Review and edit AI-generated content before committing
